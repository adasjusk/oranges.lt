Security Policy
This document is the security policy and vulnerability reporting procedure for oranges.lt.

Reporting a Vulnerability

If you discover a security vulnerability, please report it directly by creating a new issue here:
[Create a new issue on GitHub](https://github.com)

To help us investigate and fix the problem quickly, please include the following details in your report:
Vulnerability Type: (e.g., XSS, SQL Injection, configuration flaw).
Steps to Reproduce: A detailed description or a Proof of Concept (PoC) script/URL.
Impact: What access or damage an attacker could achieve through this flaw.

Guidelines for Responsible Disclosure

To protect our users and the website's availability, please read the following rules during your testing:
Do not perform Denial of Service (DoS / DDoS) attacks.
Do not access, modify, or delete any data belonging to other users.
Give us a reasonable amount of time to fix the issue before sharing any details publicly.
